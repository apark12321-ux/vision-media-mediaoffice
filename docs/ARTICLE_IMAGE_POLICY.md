# 기사 이미지 운영 정책

## 1. 기본 원칙

자동 발행 기사에는 실사형 생성 이미지를 기본 사용하지 않는다. 기본값은 기사 제목, 카테고리, 요약문을 조합한 텍스트 기반 정보형 이미지다.

이유는 다음과 같다.

- 보도사진처럼 보이는 생성 이미지는 독자 오인을 만들 수 있다.
- 인물, 상표, 건물, 제품이 포함된 이미지는 초상권, 상표권, 광고 오인 문제가 생길 수 있다.
- 자동 발행에서는 개별 이미지의 권리 확인이 어렵다.
- 정보형 이미지는 기사 내용과 결이 맞으면 독자 이해를 돕는 시각 자료로 사용할 수 있다.

## 2. 자동 발행 이미지 우선순위

1. 직접 촬영 또는 자체 제작 이미지
2. 광고주 또는 제보자가 권리 확인 후 제공한 이미지
3. CC0 또는 CC BY 등 상업적 이용이 가능한 이미지
4. Pexels, Unsplash, Pixabay 등 무료 사용이 가능한 이미지
5. 위 조건이 없으면 텍스트 기반 정보형 이미지 자동 생성

## 3. 자동 발행에서 금지하는 이미지

- 출처와 라이선스가 불명확한 이미지
- 비상업적 이용 전용 이미지
- 변경 금지 라이선스 이미지
- 인물 얼굴이 뚜렷하지만 동의 여부가 불명확한 이미지
- 상표, 로고, 간판, 제품명이 뚜렷한 이미지
- 정부 인증, 공공기관 보증처럼 오인될 수 있는 이미지
- 의료, 미용, 건강 관련 과장 효과를 암시하는 전후 비교 이미지
- 보도사진처럼 보이는 생성 이미지

## 4. 무료 이미지 사용 시 필수 메타데이터

외부 무료 이미지를 사용할 경우 articles 테이블에 아래 값을 저장한다.

- image_caption
- image_source_name
- image_source_url
- image_author
- image_license
- image_license_url
- visual_mode = photo

표시 예시:

`▲ 기사 관련 자료 이미지. 출처=Pexels, 사진=작성자명, 라이선스=Pexels License.`

## 5. 텍스트 기반 정보형 이미지 기준

이미지가 없거나 권리 확인이 애매하면 visual_mode는 auto 또는 text_card로 둔다. 이 경우 기사 상세 화면은 다음 요소로 정보형 이미지를 자동 구성한다.

- 매체명
- 카테고리
- 기사 제목
- 기사 요약
- 주요 태그

캡션은 다음과 같이 표시한다.

`▲ 기사 내용을 바탕으로 생활경제저널이 자체 제작한 정보형 이미지입니다.`

## 6. 관리자 기능 개발 시 반영할 체크리스트

- 외부 이미지를 등록하면 출처와 라이선스 입력을 요구한다.
- 출처 또는 라이선스가 비어 있으면 자동 발행을 경고한다.
- 인물/상표/민감 업종 이미지는 수동 검수 대상으로 표시한다.
- 이미지 없이 발행하면 자동으로 정보형 이미지를 표시한다.
- 포토뉴스 영역에는 실제 이미지가 있는 기사만 우선 배치하고, 없으면 정보형 썸네일로 대체한다.
