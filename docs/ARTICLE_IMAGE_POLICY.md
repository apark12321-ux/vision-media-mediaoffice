# 기사 이미지 운영 정책

## 1. 수정된 기본 원칙

자동발행 기사라도 공개 화면에는 반드시 실제 이미지가 노출되어야 한다. 기사 대표 이미지 영역에 텍스트 카드, 가짜 썸네일, AI 생성 사진처럼 보이는 임시 이미지를 기본값으로 쓰지 않는다.

대표 이미지는 아래 순서로 결정한다.

1. 관리자가 직접 업로드한 대표 이미지
2. 광고주·제보자가 권리 확인 후 제공한 이미지
3. Pexels, Unsplash, Pixabay 등 무료 사용 가능한 실제 자료사진
4. 카테고리별 기본 무료 자료사진

텍스트 기반 정보 카드는 기사 본문 보조 자료, 카드뉴스, 요약 그래픽으로는 사용할 수 있지만 `thumbnail_url` 또는 대표 이미지 fallback으로 쓰지 않는다.

## 2. 자동발행 이미지 처리 방식

자동발행기는 기사를 생성할 때 이미지를 직접 확정하지 않는다. 대신 다음 정보를 관리자 검수 대기 상태로 저장한다.

- 추천 이미지 URL
- 이미지 출처명
- 원본 출처 URL
- 저작자명
- 라이선스명
- 라이선스 URL
- 이미지 캡션
- 인물·상표·민감업종 리스크 메모

관리자가 승인한 뒤에만 `thumbnail_url`에 실제 이미지 URL을 저장한다. 승인 전에는 카테고리별 기본 무료 자료사진이 노출된다.

## 3. 금지 사항

- 텍스트 카드를 대표 이미지로 자동 노출 금지
- 보도사진처럼 보이는 AI 실사 이미지 기본 사용 금지
- 출처·라이선스가 없는 이미지 사용 금지
- 비상업 전용 이미지 사용 금지
- 인물 얼굴이 뚜렷한 이미지 자동 사용 금지
- 상표, 로고, 간판, 제품명이 뚜렷한 이미지 자동 사용 금지
- 의료·미용·건강 관련 전후 비교 이미지 자동 사용 금지
- 정부 인증, 공공기관 보증처럼 오인될 수 있는 이미지 사용 금지

## 4. articles 테이블 이미지 메타데이터

외부 무료 이미지를 사용할 경우 아래 값을 저장한다.

- thumbnail_url
- image_caption
- image_source_name
- image_source_url
- image_author
- image_license
- image_license_url
- visual_mode = photo

표시 예시:

`▲ 기사 관련 자료사진. 출처=Unsplash, 라이선스=Unsplash License.`

## 5. 관리자 화면 요구사항

기사 작성/수정 화면에는 반드시 아래 입력 영역이 있어야 한다.

- 대표 이미지 URL 또는 Supabase Storage 업로드
- 이미지 미리보기
- 이미지 캡션
- 출처명
- 원본 URL
- 저작자명
- 라이선스명
- 라이선스 URL
- 권리 확인 체크박스
- 인물/상표 포함 여부 체크박스
- 자동발행 이미지 후보 승인 버튼

발행 전 검증 규칙:

- 대표 이미지가 없으면 카테고리 기본 이미지로 대체된다는 경고 표시
- 외부 이미지인데 출처명 또는 라이선스가 없으면 발행 경고
- 인물/상표 포함 체크가 있으면 수동 검수 필요 표시
- 제휴 콘텐츠는 광고주 제공 이미지 권리 확인 체크 필수

## 6. 공개 화면 표시 원칙

- 기사 카드, 메인 편집판, 포토뉴스, 최신기사 썸네일에는 항상 실제 이미지가 들어가야 한다.
- 이미지가 없으면 카테고리 기본 무료 자료사진을 사용한다.
- 기본 자료사진은 `자료사진` 라벨을 표시한다.
- 본문 이미지 캡션에는 출처와 라이선스를 가능한 한 명시한다.

## 7. 장기 개선 방향

1. Supabase Storage 이미지 업로드
2. 무료 이미지 API 검색 후보 표시
3. 관리자 승인 후 `thumbnail_url` 확정
4. 이미지 출처/라이선스 자동 기록
5. 포토뉴스는 실제 업로드 이미지 또는 승인된 무료 이미지가 있는 기사 우선 노출
