# 생활경제저널 발행 일정 및 기사 데이터 정책

## 핵심 원칙

생활경제저널은 정기적으로 갱신되는 생활경제 전문 매체를 목표로 한다. 다만 데이터 무결성과 심사 대응 안정성을 위해 기사 날짜는 아래 원칙으로 관리한다.

1. `created_at`은 DB에 레코드가 생성된 시점을 의미한다.
2. `published_at`은 사이트에 공개 발행된 시점을 의미한다.
3. `scheduled_at`은 앞으로 공개할 예약 발행 시점을 의미한다.
4. `original_published_at`은 기존 채널에서 이미 공개되었던 콘텐츠를 이관할 때만 사용한다.
5. `imported_at`은 이관 작업을 수행한 시점을 의미한다.

## 허용되는 기사 데이터 유형

### 신규 기사

- 관리자에서 작성한다.
- 검수 후 발행한다.
- 공개 발행 순간에 `published_at`이 기록된다.

### 예약 기사

- `scheduled_at`을 입력한다.
- 상태는 `review` 또는 `scheduled`로 관리한다.
- 자동 공개 기능을 붙이더라도 관리자 검수 후 공개한다.

### 이관 기사

- 기존 블로그, 이전 사이트, 보도자료 원문 등 실제 공개 이력이 있는 콘텐츠만 이관한다.
- 이 경우 `is_imported_archive = true`, `original_published_at`, `archive_source_label`, `source_urls`를 함께 저장한다.
- 사이트 화면에서는 필요 시 “이전 발행분 이관” 또는 “자료 출처”를 표시할 수 있다.

## 금지되는 방식

- 공개된 적 없는 글을 과거에 발행된 것처럼 표시하지 않는다.
- `created_at`을 운영상 임의로 변경하지 않는다.
- 운영 DB에서 seed 데이터를 자동 실행하지 않는다.
- 출처가 없는 최신 이슈 기사 데이터를 대량 생성해 바로 발행하지 않는다.

## 초기 기사 데이터 운영 권장안

1. 심사용으로는 실제 작성·검수한 기사만 발행한다.
2. 짧은 기간에 대량 발행하기보다 일정한 편집 리듬을 유지한다.
3. 카테고리별 최소 3~5개 이상의 기사 초안을 준비한다.
4. 최신 이슈 기반 기사는 반드시 참고 출처 URL과 팩트체크 메모를 저장한다.
5. 발행 캘린더는 관리자 검수 후 예약 발행으로 운영한다.

## 기사 생성 프롬프트 사용 원칙

- AI가 생성한 기사는 초안이다.
- 제목, 수치, 정책명, 기관명, 날짜는 사람이 검수한다.
- 타 매체 문장을 복사하지 않고, 출처를 바탕으로 재구성한다.
- 제휴 콘텐츠는 기사 상단에 제휴 고지를 자동 표시한다.
