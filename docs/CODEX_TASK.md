# Codex 작업 지시서

이 저장소는 알고파트너스가 운영하는 생활경제저널 + MediaOffice MVP 초안이다.

## 목표

Next.js + Supabase 기반으로 1인 인터넷 매체 운영자가 다음 업무를 혼자 처리할 수 있게 만든다.

- 공개 매체 사이트 운영
- 기사 작성/발행
- 브랜드 인터뷰 신청 접수
- 제휴 콘텐츠 고지 자동화
- 광고주 리드 관리
- 고객 CRM
- 영업 로그
- 상품/주문/납품 관리
- AI 프롬프트 생성
- 금지 표현 및 컴플라이언스 체크

## 현재 구현된 것

- Next.js App Router 프로젝트 구조
- Tailwind 기본 디자인
- Supabase 클라이언트 구조
- 공개 사이트 메인/기사 목록/기사 상세/카테고리
- 광고·제휴 페이지
- 신청 폼 및 필수/선택 동의 체크박스
- 정책 페이지 초안
- 관리자 로그인 라우트
- 관리자 대시보드 초안
- 기사 신규 작성 초안
- 리드/고객/영업/상품/주문/납품/템플릿 목록 스켈레톤
- AI 프롬프트 생성기 초안
- 금지 표현 감지 함수
- 제휴 콘텐츠 고지 함수
- Supabase schema.sql / seed.sql

## Codex 우선 작업

### Step 1: 실행 가능성 보강

1. `npm install` 후 타입 오류와 빌드 오류를 수정한다.
2. Next.js 최신 버전에 맞춰 server action, params 타입, middleware 호환성을 점검한다.
3. Admin 페이지에 공통 레이아웃/사이드바를 적용하되 `/admin/login`은 보호 대상에서 제외한다.
4. Supabase RLS 정책과 앱 코드가 충돌하지 않는지 확인한다.

### Step 2: 관리자 CRUD 완성

1. 기사 수정/삭제/발행 상태 변경 구현
2. 리드 상세/상태 변경/고객 전환 구현
3. 고객 CRM 상세 구현
4. 상품 CRUD 구현
5. 주문/납품 상태 변경 구현
6. 템플릿 CRUD 구현

### Step 3: 컴플라이언스 기능 강화

1. 기사 발행 전 체크리스트 저장
2. 금지 표현 감지 UI 개선
3. 제휴 콘텐츠 고지 미입력 시 자동 고지 강제
4. 등록 전 금지 표현 경고
5. 수신동의 없는 영업문구 생성 시 경고

### Step 4: 운영 편의 기능

1. 검색/필터/페이지네이션
2. 대시보드 통계 개선
3. 고객별 다음 연락일 관리
4. 납품 이메일 템플릿 생성
5. CSV 가져오기/내보내기

## 절대 금지

- 실제 문자 자동 발송 구현 금지
- 수신동의 없는 광고성 메시지 자동 전송 금지
- 실제 PG 결제 연동을 MVP에서 강제하지 말 것
- 등록 전 “정식 인터넷신문”, “언론 보도 송출”, “네이버 뉴스 노출 보장” 표현 사용 금지
- 제휴 콘텐츠 고지 숨김 처리 금지
